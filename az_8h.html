◆ AZTypeFlags

enum AZTypeFlags

Class/implementation/type flags.

These are various low- and high-level flags that describe the type behaviour. Some of these (like AZ_FLAG_IMPL_IS_CLASS and AZ_FLAG_FINAL) define the type behavior. Some others (like AZ_FLAG_REFERENCE and AZ_FLAG_OBJECT) duplicate the type hierarchy information for convenience.

Enumerator
AZ_FLAG_IMPL_IS_CLASS 

Marks that an implementation is a standalone class.

It exploits the feature that classes are aligned to 8 bytes, thus if any of the lowest bits are set, the AZImplementation union contains flags and type, not a pointer to the AZClass

AZ_FLAG_INTERFACE 

Type is an interface, implementation is not class.

AZ_FLAG_BLOCK 

Type is a block, value is a pointer to the instance (not the instance itself)

AZ_FLAG_REFERENCE 

Type is a reference, value creation/destruction involves reference counting.

AZ_FLAG_BOXED 

Instance if a container of another value (_AZBoxedValue) or interface (_AZBoxedInterface) type.

AZ_FLAG_OBJECT 

A special reference type that contains a pointer to it's class.

AZ_FLAG_FINAL 

Type is final, no further subclassing is allowed.

AZ_FLAG_CONSTRUCT 

The type instances have constructor or destructor.

Subclasses should not clear this flag if set set by parent.

AZ_FLAG_ZERO_MEMORY 

Instance construction should be preceded by filling memory by zeroes.

Subclasses should not clear this flag if set set by parent. If set the type can still implement constructor - which can then rely on the instance being zero-filled.

AZ_FLAG_ABSTRACT 

Type is abstract, no instancing is allowed.

This flag is NOT propagated to subclasses.

void az_init ( void  )

Initialize type system.

If AZ_SAFETY_CHECKS is set during the compilation of the library, it is called automatically whenever any method involving types is called.

It is safe to call it more than once
